# Spec — แอปงานบ้านเก็บแต้มสำหรับครอบครัว (Home Love)

> เอกสารสรุป requirements จากการสัมภาษณ์ • ครอบครัวเดียว • เขตเวลา `Asia/Bangkok` • UI ภาษาไทย
> เวอร์ชัน 1.1 — 2026-07-23

---

## 1. ภาพรวมและเป้าหมาย

แอปช่วย **สร้างวินัยและนิสัยรับผิดชอบงานบ้าน** ให้ลูกๆ ผ่านระบบเกม (gamification):
เด็กเลือกทำงานบ้าน → ถ่ายรูปผลงาน → ส่งให้ผู้ปกครองตรวจ → ได้แต้ม/สตรีค/เหรียญ → เอาแต้มไปแลกของรางวัล

ผู้ปกครองควบคุมทุกอย่าง: ตั้งค่าเด็ก/งาน/ช่วงเวลา/ของรางวัล, ตรวจให้คะแนนคุณภาพ, อนุมัติการแลกของ, และรับอีเมลแจ้งเตือน

**หลักการออกแบบ:** ใช้งานได้ทั้งมือถือ/แท็บเล็ต/คอม (responsive PWA), ใช้ในครอบครัวเดียว, โฮสฟรี, ดูแลง่าย

## 2. ขอบเขต (Scope)

- ✅ ครอบครัวเดียว — ไม่ใช่ระบบ SaaS หลายครอบครัว ไม่มี multi-tenant
- ✅ รองรับเด็กหลายคน และผู้ปกครองหนึ่งคนขึ้นไป
- ❌ ไม่ทำ push notification หาเด็ก / ไม่ทำแอป native (เป็น PWA responsive) — ขยายทีหลังได้

## 3. สถาปัตยกรรม

**แบบ A: React frontend + Google Apps Script เป็น backend API**

```
[เด็ก/ผู้ปกครอง บนมือถือ/แท็บเล็ต/คอม]
        │  HTTPS
        ▼
[React SPA — โฮสฟรี (static, PWA, responsive, ภาษาไทย)]
        │  fetch(JSON) → doGet/doPost
        ▼
[Google Apps Script Web App = Backend API]
   ├─ auth: ตรวจ PIN เด็ก / username+password ผู้ปกครอง, ออก session token
   ├─ business logic: คำนวณแต้ม/สตรีค/เหรียญ, กติกาช่วงเวลา, จองแต้ม, LockService
   ├─ อ่าน/เขียน  ─────────►  [Google Sheet = Database]
   ├─ อัปโหลดรูป  ─────────►  [Google Drive = ที่เก็บรูป]
   └─ ส่งอีเมล    ─────────►  [Gmail (MailApp) → ผู้ปกครอง]
```

| ส่วน | เทคโนโลยี |
|---|---|
| Frontend | React + Vite, PWA, i18n ไทย, responsive |
| Backend API | Google Apps Script (`doGet`/`doPost`) |
| ฐานข้อมูล | Google Sheet (แต่ละ tab = ตาราง) |
| เก็บรูป | Google Drive |
| อีเมล | Gmail ผ่าน `MailApp` |
| โฮส frontend | ฟรี (GitHub Pages / Netlify / Vercel) |

**ความปลอดภัยสำคัญ:** logic ตรวจ PIN/รหัสผ่าน, คำนวณแต้ม และ credential ทั้งหมด **อยู่ฝั่ง Apps Script เท่านั้น** — frontend ไม่เคยถือ secret และไม่คำนวณแต้มเอง (กันเด็กโกงหรือดู PIN พี่น้อง). PIN/รหัสผ่านเก็บแบบ hash.

## 5. Data model (Google Sheet — แต่ละ tab)

### Children
`id, ชื่อ, avatar, สีประจำตัว, pinHash, แต้มคงเหลือ, สตรีคปัจจุบัน, สตรีคสูงสุด, วันล่าสุดที่ได้สตรีค, active(เปิด/ปิดใช้งาน)`

### Parents
`id, username, passwordHash, email`

### TimeWindows (ช่วงเวลา)
`id, ชื่อ, เวลาเริ่ม, เวลาสิ้นสุด, cutoff(เวลาที่ยังได้แต้มเต็ม), วันในสัปดาห์ที่เปิด, ตัวคูณโบนัส(default 1.0)`

### Chores (งานบ้าน)
`id, ชื่อ, ไอคอน, แต้มพื้นฐาน, รายการ TimeWindow ที่ทำได้, เปิด/ปิด`

### Submissions (การส่งงาน)
`id, choreId, timeWindowId, ผู้ส่ง, สมาชิกทีม[], ลิงก์รูป(Drive), เวลาส่ง, สถานะ(รอตรวจ/ผ่าน/ตีกลับ), คุณภาพ%, เหตุผลตีกลับ, แต้มที่ได้ต่อคน, ผู้ตรวจ, เวลาตรวจ`

### Rewards (ของรางวัล)
`id, ชื่อ, ราคาแต้ม, ลิมิตต่อวัน/สัปดาห์/เดือน, เปิด/ปิด`

### Redemptions (การแลก)
`id, childId, rewardId, แต้มที่จอง, สถานะ(รออนุมัติ/อนุมัติ/ปฏิเสธ), เวลาขอ, เวลาตัดสิน`

### Wishes (คำอธิษฐาน)
`id, childId, ข้อความ, เวลา, สถานะ(ใหม่/แปลงเป็นรางวัลแล้ว/ปิด)`

### Badges (เหรียญ)
`childId, ชนิด(streak-3/7/14/30), เวลาที่ได้`

### PointAdjustments (การปรับแต้มด้วยมือ)
`id, childId, delta(+/- จำนวนเต็ม), เหตุผล, ผู้ปรับ(parentId), เวลา`

### Config
`late%(default 60), team%(default 70), เขตเวลา(Asia/Bangkok), ฯลฯ`

## 6. Business logic

### 6.1 สูตรคำนวณแต้ม
คำนวณฝั่ง backend **เมื่อผู้ปกครองกด "อนุมัติ"**:

```
แต้ม = floor( แต้มพื้นฐาน × คุณภาพ% × ตัวคูณตรงเวลา × ตัวคูณโบนัสช่วงเวลา × ส่วนแบ่งทีม )

  คุณภาพ%        = 10–100% ที่ผู้ปกครองเลื่อนให้
  ตัวคูณตรงเวลา  = 1.0 ถ้าส่งก่อน cutoff, มิฉะนั้น 0.60 (late%)
  ตัวคูณโบนัส    = ของ TimeWindow (เช่น วันหยุด 1.5)
  ส่วนแบ่งทีม    = 1.0 ถ้าทำคนเดียว, 0.70 (team%) ต่อคน ถ้าเป็นทีม
```

- ปัดแต้มเป็นจำนวนเต็ม (ปัดลง)

### 6.2 ช่วงเวลา (Time Windows)
- แต่ละงานอยู่ในช่วงเวลาที่กำหนด เช่น เช้า 08:00–10:00
- ส่งก่อน **cutoff** (เช่น 09:00) = ได้แต้มเต็ม
- ส่งหลัง cutoff แต่ยังไม่หมดช่วง = ได้แต้มลด (60%)
- **หมดช่วงเวลาแล้วส่งไม่ได้** — backend ปฏิเสธการส่ง
- เปิดเฉพาะวันในสัปดาห์ที่ตั้งไว้
- ตั้งตัวคูณโบนัสต่อช่วงเวลาได้ (เช่น วันหยุด ×1.5)

### 6.3 การส่งงาน
- **บังคับแนบรูปถ่ายทุกครั้ง** (อัปโหลดขึ้น Drive)
- ทำเดี่ยว หรือ **จับคู่ทีม**: คนหนึ่งส่งงานแทนทีม แล้วเลือกเพื่อนร่วมทีม → ทุกคนในทีมได้แต้มคนละ 70%

### 6.4 สตรีค (Streak)
- นับ "ทำต่อเนื่อง" เมื่อเด็กมีงาน **ที่ผ่านการตรวจอย่างน้อย 1 งานต่อวัน**
- เมื่ออนุมัติงานแรกของวัน: วันก่อนหน้าต่อเนื่อง → +1, ขาดวัน → reset เป็น 1
- แตะ **3 / 7 / 14 / 30 วัน** → มอบเหรียญ (กันการมอบซ้ำ)

### 6.5 ของรางวัล + การแลก
- ผู้ปกครองสร้างของรางวัลพร้อมราคาแต้ม และจำกัดจำนวนครั้งต่อวัน/สัปดาห์/เดือนได้
- เด็กกดขอแลก → ตรวจแต้มพอ + ยังไม่เกินลิมิต → **หักแต้มทันที (จองไว้)** สร้าง Redemption สถานะ "รออนุมัติ"
- ผู้ปกครอง **อนุมัติ** = คงหักไว้ / **ปฏิเสธ** = คืนแต้ม

### 6.6 คำอธิษฐาน (Wishes)
- เด็กส่งข้อความบอกผู้ปกครองว่าอยากได้อะไร
- ผู้ปกครองเห็นในแอป และ **แปลงเป็นของรางวัลจริง** (สร้าง Reward) ได้

### 6.7 กันการชนกันของข้อมูล (Concurrency)
- ใช้ `LockService` ของ Apps Script ตอนหักแต้ม/อนุมัติ/จองแต้ม เพื่อกัน race condition

### 6.8 ปรับแต้มด้วยมือ (Manual Adjustment)
- ผู้ปกครอง **บวก/ลบแต้ม** ให้เด็กได้ตรงๆ นอกเหนือจากการอนุมัติงาน (เช่น ให้โบนัสพิเศษ หรือหักโทษ)
- **บังคับใส่เหตุผลทุกครั้ง** — เหตุผลว่างไม่ได้ backend ปฏิเสธถ้าไม่มีเหตุผล
- ทุกการปรับ **บันทึกลง `PointAdjustments`** (childId, delta, เหตุผล, ผู้ปรับ, เวลา) แล้วอัปเดต "แต้มคงเหลือ" ของเด็ก
- **ไม่ให้แต้มคงเหลือต่ำกว่า 0** — ถ้าลบเกินแต้มที่มี backend ปฏิเสธพร้อมข้อความแจ้ง
- ใช้ `LockService` ตอนปรับ (กันชนกับการอนุมัติงาน/จองแลกของ) — หลักการเดียวกับ 6.7
- การปรับมือ **ไม่กระทบสตรีค/เหรียญ** (สตรีคผูกกับงานที่ผ่านการตรวจเท่านั้น ตาม 6.4)
- ทำฝั่ง backend เท่านั้น — frontend ส่งเฉพาะ childId, delta, เหตุผล ไม่คำนวณแต้มเอง

## 8. หน้าจอ (Screens)

### 8.1 ฝั่งเด็ก
1. เลือกรูปตัวเอง → กด PIN
2. หน้าหลัก: แต้มสะสม, สตรีค, เหรียญรางวัล
3. เลือกงาน (กรองเฉพาะช่วงเวลาที่เปิดตอนนี้) → ทำเสร็จ → ถ่าย/แนบรูป → เลือกทำเดี่ยว/จับคู่ทีม → กดส่ง
4. ดูสถานะงาน (รอตรวจ / ผ่าน / ตีกลับพร้อมเหตุผล)
5. ร้านของรางวัล → กดแลก (รอผู้ปกครองอนุมัติ)
6. ส่งคำอธิษฐาน

### 8.2 ฝั่งผู้ปกครอง
1. **คิวตรวจงาน**: ดูรูป → เลื่อนให้คะแนนคุณภาพ 10–100% → อนุมัติ หรือตีกลับพร้อมเหตุผล
2. **คำขอแลกของ**: อนุมัติ / ปฏิเสธ
3. **คำอธิษฐาน**: ดู + แปลงเป็นของรางวัล
4. **ปรับแต้มลูก**: เลือกเด็ก → ใส่ +/- และ **เหตุผล (บังคับ)** → บันทึก (ตาม 6.8)
5. **ตั้งค่า** — **CRUD เต็ม (เพิ่ม/แก้/ลบ)** ทำเองได้ทั้งหมด:
   - **เด็ก:** เพิ่ม/แก้/ลบ (ถ้ามีประวัติแต้มให้ **ปิดใช้งาน** แทนลบ) — ชื่อ, avatar การ์ตูน, PIN 4 หลัก, สีประจำตัว
   - **ช่วงเวลา:** เพิ่ม/แก้/ลบ — เวลาเริ่ม/สิ้นสุด, cutoff, วันในสัปดาห์, ตัวคูณโบนัส
   - **งานบ้าน:** เพิ่ม/แก้/ลบ (ถ้ามีการส่งงานอ้างอิงอยู่ให้ **ปิดใช้งาน** แทนลบ) — แต้มพื้นฐาน, ช่วงเวลาที่ทำได้
   - **ของรางวัล:** เพิ่ม/แก้/ลบ (ถ้ามีการแลกอ้างอิงอยู่ให้ **ปิดใช้งาน** แทนลบ) — ราคาแต้ม, ลิมิตต่อวัน/สัปดาห์/เดือน
6. **ประวัติ/รายงาน** แต้ม (รวมประวัติการปรับแต้มด้วยมือจาก `PointAdjustments`)

## 11. Verification (ทดสอบ end-to-end)

- **Backend:** ทดสอบทุก endpoint — auth ถูก/ผิด PIN, ส่งงานในเวลา/นอกเวลา, คำนวณแต้ม (ตรงเวลา / สาย / ทีม / โบนัส) ตรงสูตร, จอง/คืนแต้ม, เพิ่มสตรีค+เหรียญ
- **ปรับแต้มมือ:** บวก/ลบแต้มพร้อมเหตุผล → บันทึกลง `PointAdjustments` + แต้มคงเหลืออัปเดตถูก; เหตุผลว่างถูกปฏิเสธ; ลบเกินจนติดลบถูกปฏิเสธ; ยืนยันสตรีค/เหรียญไม่เปลี่ยน
- **CRUD:** เพิ่ม/แก้/ลบ เด็ก/งาน/รางวัล/ช่วงเวลา ได้ครบ; กรณีมีข้อมูลอ้างอิงอยู่ต้อง **ปิดใช้งาน** แทนลบ (ประวัติ/รายงานไม่พัง)
- **Frontend:** เดินจริงตาม flow — เด็กล็อกอิน → ส่งงานพร้อมรูป → ผู้ปกครองตรวจให้คะแนน → แต้มเข้ากระเป๋า → แลกของ → อนุมัติ; ทดสอบทีม 2 คน; คำอธิษฐาน → แปลงเป็นรางวัล
- **อีเมล:** ยืนยันผู้ปกครองได้อีเมลเมื่อมีงานส่ง/ขอแลก/คำอธิษฐานใหม่
- **Data:** เปิด Google Sheet ดูว่าข้อมูลถูกเขียนครบถูกต้อง

### CI/CD (GitHub Actions)
- **Frontend:** push `frontend/**` → build+deploy GitHub Pages (secret `VITE_API_URL`)
- **Backend:** push `apps-script/**.gs` → `clasp push`+`clasp deploy` คง URL เดิม (secrets `CLASPRC_JSON`, `CLASP_SCRIPT_ID`, `CLASP_DEPLOYMENT_ID`)
- deploy backend มือ: `cd apps-script && npm run redeploy`
